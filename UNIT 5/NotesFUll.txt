## **PART 1: HEAPS**

### **1. Definition**

* A **heap** is a **complete binary tree**
* It follows a **heap property**

Types:

* **Max Heap** → Parent ≥ Children
* **Min Heap** → Parent ≤ Children

---

### **2. Properties**

* Always **filled left to right**
* Stored in **array form** (no pointers needed)

Index relations (1-based):

* Left child = `2i`
* Right child = `2i + 1`
* Parent = `i / 2`

---

### **3. Heap Operations**

#### **(A) Insertion (Heapify Up)**

Steps:

1. Insert at **end**
2. Compare with parent
3. Swap until property satisfied

Time Complexity: **O(log n)**

---

#### **(B) Deletion (Heapify Down)**

Steps:

1. Remove root
2. Replace with last element
3. Compare with children
4. Swap with correct child

Rules:

* Max heap → swap with **larger child**
* Min heap → swap with **smaller child**

Time Complexity: **O(log n)**

---

### **4. Heapify**

* Process of maintaining heap property
* Two types:

  * **Heapify Up** (insertion)
  * **Heapify Down** (deletion)

---

### **5. Applications**

* Priority Queue
* Heap Sort
* Scheduling problems

---

### **6. Important Points (Viva)**

* Heap ≠ BST
* No traversal importance
* Structure first, then order

---

## **PART 2: HASHING**

---

### **1. Definition**

* Technique to store data for **fast access**
* Uses a **hash function**

```
key → hash function → index
```

Example:
`h(x) = x % 10`

---

### **2. Collision**

* When two keys map to same index

Example:
`23 % 10 = 3`
`13 % 10 = 3`

---

## **3. Collision Resolution**

---

### **(A) Chaining**

* Each index stores a **list**

Example:

```
3 → 23 → 13
```

Pros:

* Easy insertion/deletion
* No overflow

Cons:

* Extra memory

---

### **(B) Open Addressing**

* Find another slot

---

#### **1. Linear Probing**

```
h(x) + i
```

Example:

* 23 → 3
* 33 → 4
* 43 → 5

Problem:

* **Clustering**

---

#### **2. Quadratic Probing**

```
h(x) + i²
```

Example jumps:
1, 4, 9…

Better distribution

---

#### **3. Double Hashing**

```
h(x) + i * h2(x)
```

* Uses second hash function
* Best method

---

### **4. Deletion in Hashing**

* Chaining → direct removal
* Open addressing → mark as **deleted (dummy)**

---

### **5. Load Factor (λ)**

```
λ = number of elements / table size
```

Example:
7 elements, size 10 → λ = 0.7

* Low λ → efficient
* High λ → more collisions

---

### **6. Important Points (Viva)**

* Hashing = fast access
* Collisions unavoidable
* Chaining vs Open addressing difference
* Load factor importance
